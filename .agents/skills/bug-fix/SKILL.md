# Bug Fix（事実確認重視）

## 手順

1. エラー内容やスタックトレースを精読し、関連するコード・ファイルを自分で必ず確認し、具体的な根本原因を特定する。
2. 必要な情報が不足していて根本原因が分からない場合は「何がどう不足しているか」を明示し開発者に報告する。
3. 根本原因を特定できたら、「最小限かつ有効」な修正手法を選択する。
4. 修正を適用する。
5. ローカルで必ず lint, format, build, test コマンドを実行して、型・構文・テストが全て正常に通ることを自身で確認する。
6. フロントエンドやUIに関わる修正の場合は、**Playwright等のUIテストまたは実際のブラウザ確認**も必須。自動テストだけで担保できない場合は自身で画面を開いて「直ったことを肉眼で確認」する。
7. テストまたはUI確認で失敗/未修正がまだあれば、**必ず再現手順・画面・エラーログを自分で直接確認し、1に戻ってやり直す**。
8. すべて解消したら「どのコマンドで検証し何を根拠に直ったと判断したか」を明示して完了とする。

## 注意

- 決して推測や「たぶん大丈夫」では終えず、**実際に走らせて・見て・検証できた事実を必須条件**とする。
- Playwright等のE2E/UIテストが用意されていない場合は、その旨を明記の上で手動検証し記録する。
- 修正の過程・検証状況はプルリクや報告時に必ず共有すること。
